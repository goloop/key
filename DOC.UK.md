## Ментальна модель

`key` перетворює ідентифікатор `uint64` на короткий оборотний рядковий ключ над
власним алфавітом і декодує його назад. Корисно для скорочення URL,
обфускації послідовних ID бази даних і випуску людиночитних кодів на кшталт
квитків чи купонів.

Єдиний тип, з яким ви працюєте, — це `*Locksmith`. Він:

- **Детермінований і двобічний** — кожен ID має рівно один канонічний ключ, і
  кожен валідний ключ декодується рівно в один ID.
- **Точний на всьому діапазоні `uint64`** — вся арифметика лише цілочислова
  (`uint64` з перевірками переповнення `math/bits`), тож немає дрейфу
  з рухомою комою й тихого урізання великих значень.
- **Незмінний** — `Locksmith` конфігурується раз при створенні; немає змінного
  глобального стану, який можна пошкодити.

```go
import key "github.com/goloop/key/v2"
```

## Теорія: позиційний кодек base-N

З алфавітом із N символів значення записується в системі base N — кожен символ
є цифрою. З фіксованим розміром ключа отримуєте скінченну послідовність
унікальних комбінацій, кожна з унікальним числовим індексом від `0` до
`Total-1`. За індексом (скажімо, ID бази) отримуєте ключ; за ключем відновлюєте
індекс.

Для алфавіту `"abc"` й фіксованого розміру ключа 3:

```
    0. aaa    1. aab    2. aac    3. aba    4. abb    5. abc
    6. aca    7. acb    8. acc    9. baa   10. bab   11. bac
   12. bba   13. bbb   14. bbc   15. bca   16. bcb   17. bcc
   18. caa   19. cab   20. cac   21. cba   22. cbb   23. cbc
   24. cca   25. ccb   26. ccc
```

Комбінацій 3³ = 27. Ітерація 10 відповідає `"bab"`; комбінація `"aba"` — це
ітерація 3.

## Конструктори

```go
func NewDynamic(alphabet string) (*Locksmith, error)
func NewFixed(alphabet string, size int) (*Locksmith, error)
func MustNewDynamic(alphabet string) *Locksmith
func MustNewFixed(alphabet string, size int) *Locksmith
```

`NewDynamic` будує кодек змінної довжини; `NewFixed` — фіксованої. Варіанти
`MustNew…` панікують на поганому алфавіті й призначені для пакетних кодеків зі
сталими параметрами.

Алфавіт має містити **щонайменше два унікальні символи й жодного дубліката**. На
помилці повернений `*Locksmith` дорівнює `nil`, а помилка — один із
[сентинелів алфавіту/розміру](#помилки).

```go
ls, err := key.NewFixed("abcde", 3)
coupon := key.MustNewFixed(key.Unambiguous, 6) // пакетний рівень, сталі параметри
```

## Фіксована vs динамічна довжина

**Фіксований** `Locksmith` завжди дає ключі заданої довжини, доповнені зліва
першим символом алфавіту:

```go
ls, _ := key.NewFixed("abc", 8)
ls.Marshal(10) // "aaaaabab"
```

**Динамічний** `Locksmith` дає ключі настільки короткі, наскільки дозволяє
значення:

```go
ls, _ := key.NewDynamic("abc")
ls.Marshal(1)        // "b"
ls.Marshal(10)       // "bab"
ls.Marshal(10000000) // "caacbbaabbacbab"
```

Обирайте фіксований, коли довжина ключа має бути сталою (колонки, друковані
коди, сортовні ідентифікатори); динамічний — коли хочете найкоротший ключ.

## Кодування й декодування

```go
func (l *Locksmith) Marshal(id uint64) (string, error)
func (l *Locksmith) MarshalAppend(dst []byte, id uint64) ([]byte, error)
func (l *Locksmith) Unmarshal(key string) (uint64, error)
```

`Marshal` кодує ID у ключ; повертає [`ErrIDTooLarge`](#помилки), коли ID не
вміщується у фіксований простір ключів. `MarshalAppend` кодує й дописує байти
ключа до `dst`, оминаючи алокацію рядка на гарячих шляхах. `Unmarshal` декодує
ключ назад в ID (суворо — див. нижче).

```go
ls, _ := key.NewFixed("abcde", 3)
v, _ := ls.Marshal(122)     // "eec"
i, _ := ls.Unmarshal("eec") // 122
```

## Суворе декодування й канонічні ключі

`Unmarshal` забезпечує бієкцію: ключ завжди повертається сам у себе, тож для
кожного ID є рівно одне канонічне написання. Неканонічні, порожні, неправильно
розмірені чи поза діапазоном ключі відхиляються типізованим сентинелом:

```go
ls, _ := key.NewDynamic("abc")

_, err := ls.Unmarshal("")   // key.ErrEmptyKey
_, err = ls.Unmarshal("aab") // key.ErrNonCanonical (канонічний ключ для 1 — "b")
_, err = ls.Unmarshal("abX") // key.ErrUnknownChar

ok := ls.Valid("bab") // true; Valid істинний тоді й лише тоді, коли Unmarshal успішний
```

```go
func (l *Locksmith) Valid(key string) bool
```

`Valid` — зручний предикат: істинний рівно тоді, коли `Unmarshal` був би
успішним, без алокації декодованого значення.

## Огляд Locksmith

```go
func (l *Locksmith) Alphabet() string
func (l *Locksmith) Size() uint64
func (l *Locksmith) Total() uint64
func (l *Locksmith) Saturated() bool
```

- `Alphabet` — алфавіт, з яким побудовано Locksmith.
- `Size` — фіксована довжина ключа або `0` для динамічного Locksmith.
- `Total` — кількість представних ID; валідні ID пробігають `0..Total-1`.
  Насичується до `MaxUint64` для просторів щонайменше 2⁶⁴.
- `Saturated` — чи кожен `uint64` ID вміщується в простір ключів.

```go
ls, _ := key.NewFixed("abc", 8)
ls.Size()  // 8
ls.Total() // 6561 (3^8)
```

## Ітерація

```go
func (l *Locksmith) Iter(from, to uint64) iter.Seq2[uint64, string]
func (l *Locksmith) IterN(from, n uint64) iter.Seq2[uint64, string]
```

Обидві — послідовності range-over-func, що видають пари `(id, key)`. `Iter` йде
включним діапазоном ID `[from, to]`; `IterN` видає щонайбільше `n` ключів,
починаючи з `from`, і обмежений лічильником, що безпечніше для масової генерації
біля верхньої межі простору ID.

```go
for id, k := range ls.Iter(0, 99) {
    fmt.Println(id, k)
}
for _, k := range ls.IterN(1_000_000, 50) { // 50 ключів, без ризику переповнення
    _ = k
}
```

## Випадкові ключі

```go
func (l *Locksmith) Random(r io.Reader) (string, error)
func (l *Locksmith) RandomCrypto() (string, error)
```

`Random` тягне рівномірно розподілений ID із наданого reader'а методом
відбіркового семплування, тож **немає зсуву за модулем**, і повертає його ключ.
Передайте `crypto/rand.Reader` для непередбачуваних ключів. `RandomCrypto` — це
безпечна за замовчуванням обгортка над `Random(crypto/rand.Reader)`.

```go
k, _ := ls.RandomCrypto()          // безпечно за замовчуванням
k, _ = ls.Random(mathrand.Reader)  // швидко, небезпечно — для тестів/семплування
```

## Готові алфавіти

Пакет постачає поширені алфавіти; будь-який рядок унікальних символів теж
працює:

| Константа | Алфавіт |
|-----------|---------|
| `Base16`      | `0123456789abcdef` |
| `Base32`      | `ABCDEFGHIJKLMNOPQRSTUVWXYZ234567` |
| `Base36`      | `0123456789abcdefghijklmnopqrstuvwxyz` |
| `Base62`      | `0-9 A-Z a-z` |
| `Crockford`   | Base32 Крокфорда (без `I L O U`) |
| `Unambiguous` | прибирає схожі символи для набору руками |

```go
coupon := key.MustNewFixed(key.Unambiguous, 6) // зручні для людини коди
```

## Помилки

Усі — сентинели, звіряються через `errors.Is`:

| Помилка | Виникає, коли |
|---------|---------------|
| `ErrBlankAlphabet`  | алфавіт порожній |
| `ErrShortAlphabet`  | менше двох символів |
| `ErrDuplicateChar`  | в алфавіті повторюваний символ |
| `ErrInvalidSize`    | фіксований розмір поза діапазоном |
| `ErrIDTooLarge`     | ID не вміщується у фіксований простір ключів |
| `ErrEmptyKey`       | декодовано порожній ключ |
| `ErrInvalidLength`  | фіксований ключ має неправильну довжину |
| `ErrUnknownChar`    | ключ містить символ поза алфавітом |
| `ErrNonCanonical`   | ключ — не канонічне написання свого ID |
| `ErrKeyOutOfRange`  | декодований ID поза представним діапазоном |

```go
if _, err := ls.Unmarshal(s); errors.Is(err, key.ErrNonCanonical) {
    // відхилити неканонічний ключ
}
```

## Рецепти й поради

**Обфускація послідовних ID.** Використовуйте динамічний Locksmith над `Base62`,
щоб перетворити первинний ключ бази на короткий непрозорий токен, і `Unmarshal`
на зворотному шляху:

```go
ls := key.MustNewDynamic(key.Base62)
token, _ := ls.Marshal(userID)
```

**Коди для набору руками.** Надавайте перевагу `Unambiguous` (або `Crockford`) з
`NewFixed`, щоб купони й квитки мали сталу ширину й уникали плутанини `0/O`,
`1/l`.

**Безпечні токени.** Беріть `RandomCrypto` — він тягне з `crypto/rand` без зсуву
за модулем, тож ключі водночас непередбачувані й рівномірні.

**Масова генерація.** Надавайте перевагу `IterN(from, n)` над `Iter(from, to)`
при генерації багатьох ключів біля верхньої межі простору ID; обмеження
лічильником прибирає пастку переповнення.

**Уникнення алокацій на гарячих шляхах.** `MarshalAppend` пише в повторно
використовуваний `[]byte`, що важливо при випуску ключів у щільному циклі.
